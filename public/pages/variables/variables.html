Variables in Javascript;

Which variables exists and what's their pros & cons!

"let":

Pros:
"let" variables are block-scoped; They can only be accessed within the block of code where they are declared.
They also can't be redeclared within that block, which can prevent errors. - They can however be reassigned!
Cons:
They can be harder to access, since they only exists within blocks.
They also must be declared before being used, which can be annoying.
Might not be supported in all versions of Javascript.
When to use: If you have to.

"var":

Pros:
The orignal way of declaring in Javascript and therefor supported in all versions of Javascript.
"var "is function-scoped, meaning they can be accessed within the function in which they're decleared. - Also able to be redeclared and reassigned in same scope.
Cons:
Can be hoisted, meaning they can be accessed before they're declared which can cause confusion and errors.
They're not block-scoped, so they can be accessed outside of where they're declared.
"var" variables can easily be overwritten or redeclared which can lead to unexpected behavior and errors. When to use: Never!

"const":

Pros:
"const" variables are block-scoped, same as "let".
They can't be redeclared or reassigned within their scope, which can prevent errors.
Therefore they're useful for declaring values that shouldn't be changed; constants or configuration settings, etc.
When to use: Always if you can get away with it!
